# REST API (MVP) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the HTTP API described in [`../specs/2026-03-28-rest-api-design.md`](../specs/2026-03-28-rest-api-design.md) using BunŌĆÖs `fetch` handler, Prisma, and `date-fns`, with integration tests and an updated `contracts/openapi.yaml`.

**Architecture:** Keep `createFetchHandler` in `src/api/server.ts` as the entry point; add a small **router** that matches `method + pathname` and delegates to **handlers** that receive `(req, ctx)` where `ctx` includes **`userId`**, **`householdId`**, and **`prisma`**. Resolve tenancy once per request after auth. Put **Prisma queries and invariants** (same-household meal checks, membership checks for `MealCookedBy`) in **`src/api/services/`** (or similarly named) so handlers stay thin. Map domain DTOs via existing **`src/domain/mappers`**.

**Tech Stack:** Bun, TypeScript, Prisma, PostgreSQL, **`date-fns`** (add dependency), existing `src/domain` DTOs and `planDateFromYmd` / `normalizeIngredientName`.

## File map (planned)

| Path | Responsibility |
|------|----------------|
| `src/api/server.ts` | Wire `Bun.serve`, `createFetchHandler(prisma)`, delegate to router |
| `src/api/router.ts` | Match routes; strip `/api` prefix; return 404 empty for unknown |
| `src/api/auth.ts` | Parse `Authorization` + optional `X-Dev-User-Id`; load membership; return `AuthContext` or `Response` |
| `src/api/errors.ts` | `jsonError(status, code, message?)` ŌåÆ `Response` |
| `src/api/parse.ts` | Query/body helpers: `parseYmd`, pagination `limit`/`offset`, repeated `heroIngredientId` from `URLSearchParams` |
| `src/api/handlers/*.ts` | One file per area: `me.ts`, `household.ts`, `meals.ts`, `ingredients.ts`, `day-plans.ts` |
| `src/api/services/*.ts` | Business logic + transactions: `tenancy.ts`, `meals-service.ts`, `day-plans-service.ts`, `ingredients-service.ts` |
| `contracts/openapi.yaml` | Full route + schema documentation |
| `tests/integration/api/*.test.ts` | HTTP + Prisma integration tests (pattern: `health.test.ts`, `prisma-schema.test.ts`) |
